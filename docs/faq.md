# FAQ

## What is vrOS?

vrOS is a desktop-first VR overlay system for SteamVR. It combines a Windows control center with in-VR overlays for capture, keyboard input, creator tooling, and utility surfaces.

## Do I need SteamVR?

For the VR overlay experience, yes. SteamVR is the primary runtime path for the release build.

## What is the easiest first overlay to test?

A single capture overlay. It validates the runtime, the overlay path, and basic interaction without adding extra complexity.

## Why do you keep saying "desktop-first"?

Because the fastest way to set up, debug, and support vrOS is usually from the Windows control center first. VR is the presentation layer, not the only control surface.

## What if overlays do not show up after install?

Restart SteamVR once before making deeper changes. That remains the most common first-install recovery step.

## Where do I report bugs?

Use the support center at [support.vros.cat/report-bug](https://support.vros.cat/report-bug).

## Where are the old RFCs, papers, and proposal docs?

Those were intentionally removed from public navigation during release prep so the docs center can focus on installation, use, and support.
